## Veredicto Anti-Patrones
LLM: tipografía intencional (Fraunces+Outfit), copy real en español, proporciones asimétricas. Pero `.stat .num`/`.lbl` en grid de 3 es hero-metric template clásico recoloreado.
Manual (detector no disponible en este entorno): 0 gradient-text, 0 em dashes, border-right solo como separador neutro (no side-stripe). CSS muerto: `.float-card`, `.fc1`, `.fc2` sin markup correspondiente.
Overlays visuales: no disponibles (sin automatización de navegador en esta sesión).

## Lo que Funciona
1. Tablas de precios con notas de empaque, precisas sin perder sensación artesanal.
2. Escala de picor redundante (iconos + texto), accesible.
3. Roster de clientes con fallback de iniciales para logos faltantes.

## Problemas Prioritarios
[P1] Formulario sin mensajes de error reales — solo borde rojo, sin texto ni aria-invalid. Riesgo de fuga de leads en móvil. Fix: texto de error visible + aria-describedby.
[P1] Sección de confianza sobrecargada — título+lead+3 stats+4 pilares en una pantalla. Fix: redistribuir en dos respiraciones visuales.
[P2] CSS muerto (.float-card, .fc1, .fc2) sin markup. Fix: eliminar o terminar la feature.
[P2] Regla "9 vs 50 paquetes" vive parcialmente separada de la tabla. Fix: verificar consistencia entre nota y celdas.
[P3] Animación de pulso infinita en botón WhatsApp contradice "conversión sin presión". Fix: detener tras primera interacción.

## Persona Red Flags
Rosa (compradora rápida desde celular): formulario sin mensajes de error es su mayor riesgo de abandono.
Don Memo (distribuidor que compara precios): excepción "individual = 50/100 paquetes" rompe su escaneo si no lee la nota de condiciones.

## Minor Observations
- initials() puede generar iniciales poco reconocibles para nombres cortos.
- title/meta description fijos en español pese a i18n completo; OG tags no se localizan.
- Reglas .scard-toggle/tabindex residuales sin elemento correspondiente en el markup actual.
